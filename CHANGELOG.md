# Журнал изменений

## 0.1.0 - 2026-06-11

- Создан фундамент backend-монорепозитория.
- Добавлены модули `auth`, `users`, `shops`, `patrol-points`, `patrols`.
- Добавлен идемпотентный seed-скрипт для ручной проверки API.
- Исправлена генерация Swagger-схем DTO: поля теперь имеют явные типы вместо `object`.
- Зафиксировано решение использовать аппаратный NFC UID без прошивки меток.
- Добавлен backend-сценарий настройки маршрута магазина: старт настройки, просмотр прогресса и привязка NFC UID к точке по порядковому номеру.
- Для магазинов добавлены `external_id`, `route_status`, `route_expected_points`, `route_registered_points`.
- Дефолтный API-префикс приведён к `/api/v1`.
- Добавлена замена NFC-метки контрольной точки с архивированием старой метки и историей в `nfc_tag_replacements`.
- Добавлены эталонные интервалы маршрута и инциденты обходов: `short_interval`, `long_interval`, `missed_point`.
- Добавлен endpoint `GET /api/v1/patrols/incidents` для просмотра инцидентов по магазину, сотруднику, типу и периоду.
- Добавлены JWT guard, роли, `CurrentUser` и первый Mobile API для профиля и регистрации NFC-меток маршрута с телефона.
- Добавлены mobile endpoints для маршрута обходчика, активного обхода, старта обхода, онлайн-скана и offline sync с `localId`.
- Offline sync для мобильного клиента теперь возвращает статус по каждому событию: `created`, `duplicate`, `late_sync` или `point_deactivated`.
- Конфликтные офлайн-события сохраняются с флагами `late_sync` и `point_deactivated_after_scan`, не ломая прогресс уже завершённого обхода.
- Добавлен guide `docs/mobile/api-developer-guide.md` с контрактом и практической инструкцией для Android-клиента.
- Добавлен HTTP smoke/e2e-тест mobile API: логин, роли, маршрут, старт обхода и offline sync.
- Backend-документация разложена в `docs/backend`, mobile-инструкции вынесены в `docs/mobile`.
- Авторизация переведена на постоянные человекочитаемые ключи доступа: админ создает пользователя, backend генерирует ключ, мобильное приложение входит по `accessKey`.
- Миграция ключей создает bootstrap-админа `system.admin` с ключом `ADMN-0000-0001`, если в базе еще нет администраторов.
- Создание пользователя теперь валидирует `shopId` до записи в БД и возвращает доменную ошибку вместо FK-ошибки PostgreSQL.
- Реализован домен расписаний обходов: CRUD API, роли и ограничение менеджера своим магазином, проверка пересекающихся окон.
- Mobile API возвращает доступные по локальному времени магазина расписания и принимает `scheduleId` при старте обхода.
- Для планового обхода backend проверяет активность и принадлежность расписания магазину и самостоятельно рассчитывает `dueAt`.
- Добавлена фоновая проверка дедлайнов: просроченный обход получает статус `overdue`, но может быть продолжен и завершён сотрудником.
- Добавлена связь many-to-many между пользователями и магазинами с сохранением основного `shopId` для совместимости мобильного клиента.
- Добавлен `PUT /api/v1/users/:id/shops` для замены назначений пользователя на магазины.
- Добавлен защищённый `GET /api/v1/patrols/employee/:employeeId` для просмотра истории конкретного обходчика.
- Исправлено определение следующей непривязанной точки маршрута для nullable `nfc_tag_id`, загруженного TypeORM как `null`.
- Исправлена сортировка инцидентов обходов при пагинации: QueryBuilder теперь использует имена свойств сущностей (`createdAt`, `sortOrder`, `scannedAt`, `startTime`) вместо snake_case-имен колонок.
- Добавлены backend-контракты для мобильных кнопок: отмена обхода с причиной, завершение с диалоговым отчетом и сброс настройки цифрового маршрута для старта заново.
