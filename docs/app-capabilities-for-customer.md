# Возможности Patrol System для заказчика

## Общее описание

Patrol System — мобильное приложение для контроля обходов в сети магазинов. Система позволяет настраивать маршруты обхода через NFC-метки, назначать расписания, контролировать прохождение точек, фиксировать нарушения и просматривать историю обходов по магазинам и сотрудникам.

Приложение рассчитано на три роли: администратор сети, менеджер магазина и обходчик.

## Ключевые возможности

- Вход в приложение по персональному ключу доступа.
- Разделение функций по ролям.
- Управление магазинами сети.
- Управление пользователями и их активностью.
- Назначение сотрудников на один или несколько магазинов.
- Настройка маршрутов обхода через NFC-метки.
- Замена NFC-меток без потери истории.
- Создание и управление расписаниями обходов.
- Прохождение обхода строго по маршруту.
- Контроль попыток пропуска точек.
- Фиксация времени сканирования каждой точки.
- Автоматическое завершение обхода после прохождения всех точек.
- Ручная отмена обхода с указанием причины.
- Отчет сотрудника при завершении обхода.
- История обходов по магазинам и сотрудникам.
- Просмотр нарушений и подозрительных обходов.
- Push-уведомления менеджерам о нарушениях, отменах и просрочках.
- Офлайн-очередь сканов: если связь пропала, события сохраняются на устройстве и отправляются позже.
- Crash/error logging через Firebase Crashlytics.
- OTA-обновления интерфейса без полной пересборки приложения, если не меняется native-часть.

## Возможности администратора

Администратор управляет всей сетью магазинов.

- Видит все магазины сети.
- Создает и редактирует магазины.
- Активирует и деактивирует магазины.
- Управляет всеми пользователями.
- Создает администраторов, менеджеров и обходчиков.
- Назначает пользователей на магазины.
- Может выдать новый ключ доступа пользователю.
- Может сделать пользователя неактивным.
- При смене ключа или деактивации пользователя его активные сессии отзываются.
- Может настраивать маршруты магазинов.
- Может привязывать NFC-метки к точкам маршрута.
- Может сбрасывать настройку маршрута и настроить его заново.
- Может заменять NFC-метки.
- Может управлять расписаниями обходов.
- Видит историю обходов по всем магазинам.
- Видит историю обходов по сотрудникам.
- Видит нарушения по всей сети.
- Может фильтровать и сортировать магазины, пользователей, историю и нарушения.

## Возможности менеджера магазина

Менеджер работает только с назначенными ему магазинами.

- Видит только свои магазины.
- Может настраивать маршруты своих магазинов.
- Может привязывать NFC-метки к точкам маршрута.
- Может заменять NFC-метки в своих магазинах.
- Может создавать и редактировать расписания обходов.
- Видит историю обходов по своим магазинам.
- Видит историю обходов сотрудников своих магазинов.
- Видит нарушения только по своим магазинам.
- Получает push-уведомления о нарушениях своего магазина.
- Получает уведомления о просроченных обходах.
- Получает уведомления об отмене обхода сотрудником.
- Не получает данные по чужим магазинам.

## Возможности обходчика

Обходчик проходит маршруты в своем магазине.

- Видит активный обход, если он уже начат.
- Видит доступные и ближайшие плановые обходы.
- Видит день недели и время ближайшего обхода.
- Может начать обход только в разрешенное окно расписания.
- Видит кнопку обновления обходов.
- Сканирует точки маршрута через NFC.
- Проходит точки строго по порядку.
- Если пытается отсканировать точку не по порядку, приложение показывает предупреждение и не засчитывает скан.
- Такая попытка фиксируется как нарушение "пропуск точки".
- Видит прогресс обхода: сколько точек пройдено из общего количества.
- Видит текущую точку, уже отмеченные и следующие точки.
- При большом маршруте список точек прокручивается, чтобы текущая точка оставалась видимой.
- Если NFC отключен, приложение просит включить NFC.
- Во время сканирования показывается отдельный экран/оверлей сканирования.
- Может отменить обход с указанием причины.
- После прохождения всех точек может оставить отчет по обходу.
- Может продолжить активный обход после выхода из приложения.
- При отсутствии связи сканы сохраняются локально и отправляются после восстановления сети.

## Контроль нарушений

Система фиксирует несколько типов проблем:

- Попытка пропустить точку маршрута.
- Слишком быстрое прохождение между точками.
- Слишком долгое прохождение между точками.
- Просроченный обход.
- Отмена обхода сотрудником.
- Поздняя синхронизация офлайн-событий.
- Скан деактивированной точки после изменения маршрута.

Нарушения доступны в отдельном разделе с поиском, фильтрами и сортировкой.

## Расписания обходов

- Для каждого магазина можно создавать расписания обходов.
- Расписание включает название, дни недели и временное окно.
- Обходчик видит ближайшие плановые обходы.
- Если сегодня вторник, а обход назначен на среду и четверг, приложение покажет ближайший день — среду.
- Расписания сортируются по ближайшему времени.
- Активные расписания отображаются выше отключенных.
- Уже созданная история обходов сохраняется даже после отключения расписания.

## История и отчетность

- История обходов по магазину.
- История обходов по конкретному сотруднику.
- Детальная карточка обхода.
- Статус обхода: в процессе, завершен, отменен, просрочен.
- Время начала и завершения.
- Количество пройденных точек.
- События сканирования NFC.
- Причина отмены.
- Отчет сотрудника при завершении.
- Связанные нарушения.

## Безопасность и контроль доступа

- Вход по персональному ключу доступа.
- Доступ ограничен ролью пользователя.
- Менеджер видит только свои магазины.
- Обходчик работает только со своим маршрутом.
- При деактивации пользователя его выбрасывает из системы.
- При смене ключа доступа старые сессии отзываются.
- NFC UID нормализуется и проверяется на backend.
- Одна активная NFC-метка не может быть привязана к двум активным точкам одновременно.

## Технические преимущества

- Работает на Android.
- Использует реальные NFC-метки.
- Поддерживает push-уведомления.
- Поддерживает офлайн-сценарий прохождения обхода.
- Имеет журнал ошибок через Firebase Crashlytics.
- Поддерживает обновления через EAS Update.
- Backend хранит историю, нарушения, расписания, пользователей, магазины и события обходов.
