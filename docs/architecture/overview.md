# Архитектура Patrol System

Patrol System — это монорепозиторий на TypeScript с backend на NestJS, веб-панелью на React, Android-приложением на Expo и общими API-контрактами в `packages/shared`.

Backend отвечает за аутентификацию, магазины, сотрудников, NFC-точки патрулирования, события патрулирования, отчёты, уведомления и записи аудита. PostgreSQL является источником истины, Redis хранит краткоживущее операциональное состояние, такое как refresh-токены и очереди, а Swagger предоставляет описание API по адресу `/api/v1/docs`.

## Backend

Backend следует строгой модульной структуре по доменам:

- `auth` — отвечает за вход по JWT и жизненный цикл refresh-токенов.
- `users` — управляет сотрудниками, менеджерами и администраторами.
- `shops` — управляет магазинами и их организационной структурой.
- Будущие модули будут охватывать точки патрулирования, патрули, отчёты и уведомления.

Каждый домен следует схеме `Controller -> Service -> Repository`. Контроллеры преобразуют HTTP-запросы в DTO, сервисы содержат бизнес-логику и выбрасывают доменные ошибки, репозитории изолируют доступ через TypeORM.

## База данных

Начальная миграция отражает `docs/backend/schema.sql`:

- UUID первичные ключи через `gen_random_uuid()`.
- `TIMESTAMPTZ` для всех столбцов с временными метками.
- Мягкое удаление через nullable-поле `deleted_at`.
- Принудительное приведение NFC UID к нижнему регистру на уровне базы данных.
- Таблицы аудита для refresh-токенов и бизнес-действий.

## Конфигурация

Конфигурация во время выполнения загружается через `@nestjs/config` с валидацией Joi. Сервисы используют типизированные значения `ConfigService` и не читают переменные окружения напрямую.
